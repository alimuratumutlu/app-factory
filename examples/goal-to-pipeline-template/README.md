# Goal-to-Pipeline Template

This template models the new core product flow:

1. Collect a messy user goal.
2. Ask fixed intake questions.
3. Generate follow-up questions with AI.
4. Reflect the understood problem before proposing a solution.
5. Decompose the goal into milestones, actions, risks, checkpoints, and views.

The pipeline is a planning object first. Some nodes may become executable later,
but the first value is helping the user think, review, and edit the path forward.
